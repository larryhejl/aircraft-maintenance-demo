Developer Guide

Technical reference for developing and extending the Aircraft Maintenance Dashboard.

Architecture

FilePurpose
index.htmlPage structure, tabs, containers
styles.cssAll styling
script.jsLogic: data generation, filtering, rendering, charts
tests.jsIn-browser unit tests
maintenance-data.jsonStatic reference data (not loaded at runtime)

No build step required — open index.html in a browser or deploy to GitHub Pages.

Repository

  • Hosting: GitHub Pages (static deploy from main branch)
  • Live URL: https://larryhejl.github.io/aircraft-maintenance-demo/
  • Deployment: Automatic on push to main
  • Branch strategy: Direct push to main for simplicity

Data Flow

createDemoEvents()
    │
    ▼
maintenanceEvents (120 events)
    │
    ├── populateFilters() → dropdown options
    ├── filterEvents(events, filters) → filtered subset
    │       ├── renderTable(filtered)
    │       ├── renderChart(filtered, facet)
    │       └── renderStatusPie(filtered)
    └── Filter/facet change → re-runs pipeline

Function Reference

createDemoEvents() → Event[]

Generates 120 randomized events spanning one year.

parseEventDate(event) → Date

Converts YYYY-MM-DD string to UTC Date.

filterEvents(events, filters) → Event[]

Filters by aircraft, eventType, system, status, dateRangeDays.

groupEventsByMonth(events) → {month, count}[]

Buckets events by YYYY-MM, returns sorted array.

populateFilters(events) → void

Populates filter dropdowns with unique values.

renderTable(events) → void

Renders event rows, attaches click handlers for detail view.

renderChart(events, facet) → void

SVG multi-series line chart split by facet.

renderStatusPie(events) → void

SVG donut chart of status distribution.

initializeDashboard() → void

Entry point: wires tabs, filters, listeners, initial render.

Event Schema

{
    id: number,           // 1–120
    aircraft: string,     // "F-16C" | "F-15E" | "C-130J"
    tailNumber: string,   // e.g. "89-2034"
    system: string,       // "Engine" | "Avionics" | "Hydraulics" | ...
    eventType: string,    // "Unscheduled Repair" | "Scheduled Inspection" | "Time Change"
    status: string,       // "Complete" | "In Work" | "Awaiting Parts"
    date: string,         // "YYYY-MM-DD"
    hoursDown: number,    // 4–47
    description: string
}

Test Contract

window.runAllTests = function() {
    return {
        total, passed, failed, coveragePercent,
        results: [{ name, status, durationMs, details? }]
    };
};
Tests run synchronously in-browser. The dashboard UI renders metrics from the return value.

Extension Patterns

New Filter

  1. Add <select> in #controls
  2. Populate in populateFilters()
  3. Add condition in filterEvents()
  4. Wire change listener in initializeDashboard()

New Tab

  1. Add <button class="tab-button" data-tab="name"> to #tab-bar
  2. Add <div id="name-tab" class="tab-panel">
  3. Tab logic handles activation automatically

Known Limitations

  • Demo data regenerates on each page load
  • No persistence between sessions
  • Client-side only
  • SVG/CSS charts — limited interactivity
  • Minimal accessibility
  • Date filtering relative to client clock
