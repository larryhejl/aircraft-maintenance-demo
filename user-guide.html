User Guide

Overview

The Aircraft Maintenance Dashboard provides an interactive view of maintenance events for military aircraft. Use filters, charts, and the data table to explore scheduled inspections, unscheduled repairs, component replacements, and operational checks.

Getting Started

  1. Open index.html in any modern browser (Chrome, Edge, Firefox, Safari).
  2. The dashboard loads automatically with sample data.
  3. Use the tabs at the top to switch between the Dashboard view and Unit Tests.

Using Filters

The control bar at the top of the Dashboard tab provides four filters:

FilterDescription
AircraftFilter by specific aircraft tail number (e.g., F-15A #101)
Event TypeFilter by maintenance category (Inspection, Repair, Replacement, Check)
SystemFilter by aircraft system (Avionics, Engine, Hydraulics, etc.)
StatusFilter by event status (Completed, Pending, In Progress, Overdue)
Tip: Select "All" in any filter to remove that constraint. Filters combine — selecting both an aircraft and a status shows only matching events.

Reading the Data Table

The main table displays filtered maintenance events with these columns:

  • Date — When the maintenance event occurred or is scheduled
  • Aircraft — The tail number and type
  • Event Type — Category of maintenance
  • System — Aircraft subsystem involved
  • Description — Brief summary of the work
  • Status — Current state of the event

Click any row to view full event details in the detail panel on the right.

Understanding the Charts

Events Over Time (Bar Chart)

The stacked bar chart shows event counts grouped by month. Use the "Split by" dropdown to facet the chart by Aircraft, System, Event Type, or Status.

Status Breakdown (Pie Chart)

The pie chart shows the proportion of events in each status category, giving a quick health overview of the maintenance program.

Running Unit Tests

  1. Click the Unit Tests tab.
  2. Click Run Unit Tests.
  3. Review the pass/fail donut, coverage ring, and detailed results table.
Tip: Failing tests indicate code improvement opportunities. Read the "Details" column for specific recommendations.

Troubleshooting

IssueSolution
Page is blankEnsure all files (index.html, script.js, styles.css, tests.js) are in the same directory.
Charts don't renderCheck browser console for JavaScript errors. Ensure you're using a modern browser.
Filters show no optionsThe data loads on page init. Try refreshing the page.
Tests show "Not run yet"Click the "Run Unit Tests" button on the Unit Tests tab.
Note: This application uses sample data for demonstration purposes. It does not connect to any live maintenance systems.
